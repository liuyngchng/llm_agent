# 客服系统用户手册
## 1. 系统概述
本系统是一个智能客服系统，支持AI自动服务和人工客服协同工作。系统能够识别用户问题类型，并提供相应的服务，包括在线支付、个人信息处理、数据查询、人工转接和上门服务等。

## 2. 系统功能
### 2.1 消息处理流程
用户发送消息：用户通过前端界面或API发送消息

消息分类：系统自动对消息进行分类（共5种标签）

路由处理：根据分类结果将消息路由到相应的处理模块

响应返回：生成相应的响应返回给用户

### 2.2 支持的服务类型
服务类型	标签ID	说明
在线支付服务	label0	处理支付相关请求，显示支付二维码
上门服务	label1	安排上门服务，收集用户信息
个人信息处理	label2	处理个人信息提交和验证
数据检索	label3	查询用户相关数据信息
人工客服转接	label4	将对话转接给人工客服
## 3. 用户操作指南
### 3.1 访问系统
访问首页：通过浏览器访问系统根路径 /

登录认证：系统会自动重定向到登录页面

进入客服界面：登录后进入客服对话界面

### 3.2 发送消息
方式一：Web界面
在聊天框中输入问题

点击发送按钮

等待系统响应

方式二：API调用（测试用）
bash
curl -X POST 'http://127.0.0.1:19000/usr/ask' \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d 'msg=您好，我想查询我的订单&uid=用户ID'
### 3.3 接收消息



```
轮询获取消息
bash

# 定期轮询获取新消息

GET /msg/box/<uid>
```



## 4. 特殊功能说明
### 4.1 人工客服模式
请求人工客服：

发送"转人工客服"或相关关键词

系统会将当前对话历史转给人工客服

AI服务暂时关闭，等待人工客服介入

人工客服操作：

人工客服可以直接回复用户消息

人工客服可以发送"结束服务"来切换回AI模式

人工客服的回复会直接发送给用户

### 4.2 上门服务预约
当用户请求上门服务时：

系统会显示一个预约表单

自动填充已收集的用户信息

用户确认后提交预约请求

### 4.3 数据查询
系统支持查询以下类型的数据：

订单信息

账户信息

服务记录

其他用户相关数据

## 6. 系统特点
### 6.1 上下文记忆
系统会记住最近的对话历史（最多19条）

基于上下文提供更准确的回答

### 6.2 智能路由
自动识别用户意图

准确路由到相应服务模块

### 6.3 无缝切换
AI和人工客服之间可以无缝切换

对话历史在不同模式间保持连贯

## 7. 故障排除
### 7.1 常见问题
- 无法发送消息

  检查网络连接

  确认用户ID有效

  检查系统状态

- 接收不到回复

  确认轮询频率适当

  检查消息队列状态

  查看系统日志

- 分类不准确

  使用更明确的关键词

  重新描述问题

  联系技术支持

### 7.2 技术支持
如遇问题，请联系：

- 系统管理员
- 技术支持团队
- 查看系统日志文件

## 8. 注意事项
个人信息安全：不要在公共场合泄露个人敏感信息

会话超时：长时间无操作可能导致会话过期

数据存储：对话历史会根据策略定期清理

服务时间：人工客服可能有特定的服务时间

## 9. 附录
### 9.1 API接口汇总

| No.  | 接口             | 方法 | 说明         |
| ---- | ---------------- | ---- | ------------ |
| 1    | /                | GET  | 系统首页     |
| 2    | /usr/ask         | POST | 提交用户问题 |
| 3    | /msg/box/<uid>   | GET  | 获取用户消息 |
| 4    | /static/<file>   | GET  | 获取静态资源 |
| 5    | /webfonts/<file> | GET  | 获取字体文件 |

| ---- | ---- | ---------- |
